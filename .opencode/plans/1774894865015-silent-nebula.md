# Website Clarity Pass Plan

## Goal
Simplify marketing and conversion copy so non-technical visitors understand value quickly.

User decision: apply **strong simplify** on non-doc pages. Remove or replace technical/internal phrasing (for example, “public installers,” “runtime,” “entitlements,” “activation windows”) unless it is strictly required for a purchase/action step.

## Non-Negotiables
- Keep founder/marketing references as **Anthony**.
- Keep legal/rights references as **Anthony Cueva**.
- Keep socials unchanged:
  - X: `https://x.com/cuevaio`
  - LinkedIn: `https://linkedin.com/in/cuevaio`
  - Instagram: `https://instagram.com/cueva.io`
- Keep product truths intact (one-time pricing, private/local processing, app + CLI availability) in plain language.

## Plain-Language Guardrails
- Prefer: `download`, `buy once`, `works on your device`, `private processing`, `process many images`, `desktop app`, `command line`.
- Avoid on non-doc pages: `public installers`, `license-gated runtime`, `entitlements`, `surface-based licensing`, `activation window`, `grace window`, `matching keys`, `offline-after-activation`.
- Keep explanations benefit-first; only include technical mechanism when directly needed for a step.
