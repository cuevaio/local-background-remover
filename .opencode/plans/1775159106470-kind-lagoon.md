# Desktop Import UX Plan

## Goal
- Fix macOS Finder local-file drag and drop in the desktop app.
- Add `Cmd+V` support for clipboard images.
- Keep scope narrow: support Finder/local files and pasted clipboard image data only, while leaving URL import as the existing text-input flow.

## Root Cause
- `apps/desktop/src/renderer/renderer.ts` currently extracts dropped paths from `(file as File & { path?: string }).path` inside `droppedFilePaths(...)`.
- In this app, the renderer runs with `contextIsolation: true`, `nodeIntegration: false`, and `sandbox: true` (`apps/desktop/src/main/main.ts`), so relying on `File.path` is brittle.
- When Finder drops do not expose `file.path`, the renderer returns no paths and shows `"Drop local image files, or use an image URL."` before the import IPC flow is reached.

## Recommended Changes
1. Update `apps/desktop/src/preload/preload.ts` to expose a small helper that resolves local paths from dropped `File` objects using Electron-supported APIs instead of `File.path`.
2. Update `apps/desktop/src/renderer/global.d.ts` to type the new preload helper on `window.rmbg`.
3. Update `apps/desktop/src/renderer/renderer.ts` so the drop handler:
- collects dropped `File` objects from `event.dataTransfer.files`
- also checks `event.dataTransfer.items` for `kind === "file"` and calls `getAsFile()` when available
- asks preload to resolve real filesystem paths
- filters empty results and deduplicates paths before calling `window.rmbg.libraryImportFiles({ paths })`
4. Keep `apps/desktop/src/main/main.ts` unchanged unless manual testing proves an additional IPC bridge is required. The import pipeline already normalizes paths, checks file existence, and copies files correctly once it receives valid paths.
5. Add a document-level `paste` handler in `apps/desktop/src/renderer/renderer.ts` that works anywhere in the licensed workspace, except while focus is inside editable controls.
6. Add a new clipboard-image import bridge in `apps/desktop/src/preload/preload.ts` and `apps/desktop/src/renderer/global.d.ts` so the renderer can send pasted image bytes and MIME type to main safely.
7. Add a new clipboard-image import IPC path in `apps/desktop/src/main/main.ts` that:
- enforces the desktop license check
- validates non-empty image bytes
- maps supported image MIME types to file extensions
- writes the pasted image into the existing library originals directory
- returns the new `LibraryItem` through the same `items`/`imported` response shape used by current imports
8. Reuse the existing post-import behavior in the renderer so pasted images are added to the library, selected, and auto-processed like picker/URL imports.

## Files To Modify
- `apps/desktop/src/preload/preload.ts`
- `apps/desktop/src/renderer/global.d.ts`
- `apps/desktop/src/renderer/renderer.ts`
- `apps/desktop/src/main/main.ts`

## Clipboard UX Rules
- Paste works anywhere in the visible workspace, not on the onboarding/license screen.
- Paste is ignored while typing in `input`, `textarea`, `select`, or `[contenteditable]` elements.
- Only actual clipboard image data is supported in this pass.
- Pasted plain text, HTML, or URLs should not trigger imports.
- If multiple clipboard items exist, import the first image item only.
- If clipboard contains no image, do nothing silently.

## Edge Cases
- Same Finder drag appearing through both `files` and `items`: dedupe before import.
- Synthetic/web drags resolving to no local path: keep the existing invalid-drop message.
- Folder/alias drops: should fail safely without crashing; no scope expansion needed.
- Non-image local files: current behavior is unchanged unless we decide to add explicit extension validation later.
- Clipboard paste should not hijack normal typing or pasting into the URL/license inputs.
- Prevent duplicate clipboard imports from one paste event.
- Unsupported or empty clipboard image payloads should fail with a clear error only after an actual image paste attempt.
- Imported clipboard images should stay in the library even if auto-processing fails afterward.

## Verification
1. Run `cd apps/desktop && bun run lint`.
2. Run `cd apps/desktop && bun run test`.
3. Run `cd apps/desktop && bun run dev`.
4. Manual smoke test on macOS:
- drag one Finder image (`.png`, `.jpg`, `.jpeg`, `.webp`) into the dropzone and confirm it imports
- drag multiple Finder images and confirm all import
- confirm `Select files` still works
- drag a browser image/URL and confirm it is still not treated as a local-file drop
- optionally drag a folder or alias and confirm the app does not crash
5. Manual clipboard smoke test on macOS:
- copy an image, click a non-input area in the workspace, and press `Cmd+V`
- confirm a new library item appears, becomes active, and auto-processes
- focus the URL or license inputs and press `Cmd+V`; confirm no library import happens there
- copy plain text and press `Cmd+V` in the workspace; confirm nothing imports
- paste repeatedly and confirm one paste action does not create accidental duplicate imports
