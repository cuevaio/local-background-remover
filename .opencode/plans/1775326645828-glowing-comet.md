# Add Input Options Section To Homepage

## Goal
Add a new landing-page section in `apps/web` that explains the desktop app input workflow and shows the provided product GIF.

The section should communicate, explicitly:
- file selection from disk
- image import from URL
- `Cmd+V` to paste an image
- automatic background removal when an image is added
- approximate speed of `1.2 ~ 1.5 seconds per image`
- browsing previous images in the gallery
- `Cmd+C` copying the selected processed image to the clipboard for use in Canva, Photoshop, Illustrator, WhatsApp, and similar apps

Important framing:
- present this as a **desktop app** capability section
- do not imply the web app itself performs image editing in-browser

## Files To Modify
- `apps/web/src/app/page.tsx`
- `apps/web/src/components/marketing/InputOptionsSection.tsx` (new)

Optional, only if needed for cleaner asset wiring:
- `apps/web/public/...` for the provided GIF asset

## Recommended Approach
1. Create a new homepage section component at `apps/web/src/components/marketing/InputOptionsSection.tsx`.
2. Import that component into `apps/web/src/app/page.tsx` and render it immediately after `<ProofStrip />` and before the workflow comparison section.
3. Keep the section self-contained instead of introducing shared content files unless the asset path needs a tiny local constant.

## Section Structure
Use the same homepage section shell already established in `page.tsx`:
- wrapper: `section-block section-divider`
- heading stack with `Badge`, `section-title`, and muted supporting copy
- card-based content using existing `Card` UI patterns

Recommended layout:
- desktop: two-column grid
- left column: heading, supporting copy, and 4 compact feature cards
- right column: bordered GIF card using the same `bg-card/95` visual treatment as the rest of the page
- mobile: stack intro copy, then GIF, then feature cards

## Copy Framing
Recommended badge:
- `Desktop app workflow`

Recommended title:
- `Start from a file, a URL, or your clipboard`

Recommended supporting copy direction:
- explicitly mention the macOS desktop app
- include the three input methods: file, image URL, clipboard paste
- state that background removal starts automatically when an image is added
- include the approximate `1.2 ~ 1.5 seconds per image` timing as guidance, not as a strict guarantee
- mention browsing earlier results in the gallery
- mention `Cmd+C` copying the selected processed image into other tools

Recommended feature-card breakdown:
- `Choose a source`: file picker, image URL, or `Cmd+V`
- `Auto-process`: starts removal as soon as the image is added
- `Browse your gallery`: revisit earlier images in the local gallery
- `Copy into other tools`: `Cmd+C` copies the selected processed image for downstream apps

## GIF Integration
Plan to embed the provided GIF directly in the new section.

Implementation notes:
- do not process or inspect the heavy GIF during planning
- wire the GIF from `public/` once the asset path is known during implementation
- prefer a plain `<img>` for the animated GIF so integration stays simple
- keep the asset reference local to the section unless multiple sections will reuse it

## Implementation Notes
- follow the existing homepage style in `apps/web/src/app/page.tsx` and the marketing components under `apps/web/src/components/marketing/`
- keep the feature list inline in `InputOptionsSection.tsx` as a local `const`; it is homepage-only content
- use existing UI primitives: `Badge`, `Card`, and possibly `Button` only if a CTA is needed after seeing the actual composition
- avoid introducing a new pattern if the current card/grid language is sufficient

## Risks To Avoid
- do not describe this as an online editor or browser workflow
- do not say generic `URL`; say `image URL`
- do not imply `Cmd+V` works for arbitrary clipboard content; it should be framed as pasting an image
- do not imply `Cmd+C` copies any image; it should be framed as copying the selected processed image
- do not overstate the timing as a guaranteed SLA
