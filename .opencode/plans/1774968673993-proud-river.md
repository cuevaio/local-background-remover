# Plan: Fix mac notarization failure from unused updater frameworks

## Diagnosis
- `v0.4.4` no longer fails on the bundled Python runtime; the app now packages without the embedded `rmbg` fallback.
- The current notarization failure is on nested Electron updater-framework binaries:
  - `Squirrel.framework/.../Resources/ShipIt`
  - `Mantle.framework/.../Mantle`
- The desktop app does not use Electron auto-update on macOS:
  - no `electron-updater`
  - no `autoUpdater`
  - no update feed/publish config
- Desktop runtime updates are handled externally via `https://local.backgroundrm.com/install`, so these updater-related bundle contents are incidental, not product-critical.
- User decision: prefer removing the unused updater/update-delivery path first, instead of preserving it and adding more signing complexity.

## Recommended Implementation
1. Switch mac desktop packaging from `dmg + zip` to `dmg` only.
- In `apps/desktop/electron-builder.yml`, change `mac.target` to only `dmg`.
- In `apps/desktop/scripts/dist-mac.mjs`, change the builder args from `--mac dmg zip` to `--mac dmg`.

2. Update the release workflow and docs to match DMG-only desktop delivery.
- In `.github/workflows/release.yml`, stop uploading and checksumming desktop `*.zip` artifacts.
- In `apps/desktop/README.md` and `DEPLOYMENT_RUNBOOK.md`, update expected mac release assets to DMG-only.
- User decision: implement only this DMG-only change first, and add pruning only if notarization still fails afterward.

3. Re-run a signed release build before adding any extra signing hooks.
- Verify whether DMG-only packaging removes the unused updater stack from `Local Background Remover.app/Contents/Frameworks`.
- If the updater frameworks are gone and notarization passes, stop there.

4. Fallback only if the updater frameworks still appear in the app bundle.
- Add an `afterPack` hook in `apps/desktop/electron-builder.yml` pointing to a new script such as `apps/desktop/scripts/prune-mac-updater-frameworks.mjs`.
- In that script, remove the unused Squirrel.Mac framework stack before signing/notarization, including:
  - `Squirrel.framework`
  - `Mantle.framework`
  - any companion updater framework shipped alongside them
- Keep the hook mac-only and fail loudly if expected framework paths move.

5. Final fallback only if pruning is not viable.
- Add an explicit `afterSign` script to re-sign nested updater binaries/frameworks in dependency order.
- Treat this as the contingency path, not the first implementation.

## Why This Is Safe
- The desktop app already handles runtime install/update outside the Electron app via the public `rmbg` installer.
- The repo has no in-app mac auto-update code or release feed that depends on Squirrel/Mantle.
- Shipping DMG-only aligns the distributed artifact set with actual product behavior.

## Verification
1. Run `cd apps/desktop && bun run dist:mac` and confirm only `local-background-remover-vX.Y.Z-darwin-arm64.dmg` is produced.
2. Inspect `apps/desktop/dist-packages/mac-arm64/Local Background Remover.app/Contents/Frameworks` and confirm updater frameworks are absent; if not, verify the prune hook removed them before signing.
3. Re-run the signed GitHub release workflow and confirm `build-desktop` passes notarization.
4. Validate the signed app and DMG:
   - `codesign --verify --deep --strict --verbose=2 <App.app>`
   - `spctl --assess --type execute --verbose=4 <App.app>`
   - `xcrun stapler validate <App.app>`
   - `xcrun stapler validate <dmg>`
5. On a clean macOS machine, install from the DMG, launch the app, confirm runtime install still comes from `https://local.backgroundrm.com/install`, and process one image end-to-end.
