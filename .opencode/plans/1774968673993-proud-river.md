# Plan: Fix desktop release Electron version resolution

## Diagnosis
- The `build-desktop` release job reaches `apps/desktop` packaging and fails inside `electron-builder`, not during Bun or `uv` setup.
- `apps/desktop/package.json` currently declares `electron` as `^41.1.0`.
- During `bun run dist:mac`, `electron-builder` is not finding a usable installed Electron module from the desktop workspace layout, then falls back to project metadata.
- `electron-builder` rejects the ranged version string and aborts with `Cannot compute electron version ... version ("^41.1.0") is not fixed in project.`
- `apps/desktop/electron-builder.yml` does not currently set `electronVersion`, so packaging depends on version inference.

## Recommended Implementation
1. Pin the desktop Electron dependency to an exact version in `apps/desktop/package.json`.
- Change `electron` from `^41.1.0` to `41.1.0` so the project metadata itself is sufficient for packaging.

2. Add an explicit `electronVersion: 41.1.0` entry to `apps/desktop/electron-builder.yml`.
- This keeps packaging deterministic even if Bun hoists dependencies in a way that prevents `electron-builder` from resolving the installed module from `apps/desktop`.

3. Refresh the Bun lockfile after the package change.
- Keep `bun.lock` aligned so `bun install --frozen-lockfile` continues to match the desktop workspace metadata used by the release job.

4. Leave the workflow and packaging script unchanged in this pass.
- `.github/workflows/release.yml` already gets far enough to hit the packaging step.
- `apps/desktop/scripts/dist-mac.mjs` is already invoking `electron-builder` from the correct app directory with the intended unsigned-build fallback.

## Files To Modify
- `apps/desktop/package.json`
- `apps/desktop/electron-builder.yml`
- `bun.lock`

## Verification
1. From the repo root, run `bun install` to refresh `bun.lock`.
2. Re-run `bun install --frozen-lockfile` to mirror the release workflow.
3. Run `cd apps/rmbg && bun run package:mac` to stage the packaged CLI runtime.
4. Run `cd apps/desktop && bun run dist:mac` to exercise the failing path locally.
5. Confirm `apps/desktop/dist-packages/` contains the expected `.dmg` and `.zip` artifacts.
6. Re-run the release workflow on the `v0.4.1` tag or a fresh test tag after the fix lands.
