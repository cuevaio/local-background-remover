# Plan: Signed and notarized macOS desktop releases

## Diagnosis
- The current Gatekeeper dialog is expected from the repo's current release posture, not a runtime bug.
- `apps/desktop/scripts/dist-mac.mjs` disables signing when Apple credentials are absent by setting `CSC_IDENTITY_AUTO_DISCOVERY=false` and passing `--config.mac.identity=null`.
- `apps/desktop/electron-builder.yml` still has `mac.hardenedRuntime: false`, so the app is not configured for notarization.
- `.github/workflows/release.yml` builds and publishes desktop artifacts without wiring any signing or notarization secrets into the desktop job.
- `apps/desktop/README.md` and `DEPLOYMENT_RUNBOOK.md` still describe unsigned GitHub releases as acceptable, which matches the warning users now see.

## Recommended Implementation
1. Convert tagged desktop releases from unsigned fallback to fail-closed signed releases.
- Keep unsigned packaging available for local ad hoc builds.
- In `apps/desktop/scripts/dist-mac.mjs`, detect release/CI packaging and fail immediately when required Apple signing or notarization env vars are missing instead of silently shipping `identity=null` artifacts.

2. Enable Electron Builder settings required for notarization.
- In `apps/desktop/electron-builder.yml`, set `mac.hardenedRuntime: true`.
- Add explicit mac entitlements and inherited entitlements paths.
- Keep the existing `appId` stable and continue targeting `Developer ID Application`, not Mac App Store packaging.

3. Add the missing macOS signing assets.
- Add `apps/desktop/build/entitlements.mac.plist`.
- Add `apps/desktop/build/entitlements.mac.inherit.plist`.
- Keep these minimal and aligned with Electron's standard notarization requirements.

4. Wire signing and notarization secrets into the release workflow.
- In `.github/workflows/release.yml`, pass the desktop build step the required env vars:
  - `CSC_LINK`
  - `CSC_NAME`
  - `CSC_KEY_PASSWORD`
  - `APPLE_ID`
  - `APPLE_APP_SPECIFIC_PASSWORD`
  - `APPLE_TEAM_ID`
- Do not upload desktop artifacts if the signed/notarized build fails.

5. Add CI verification for signed artifacts before publishing the release.
- After `bun run dist:mac`, verify the generated app or installer with:
  - `codesign --verify --deep --strict --verbose=2`
  - `spctl --assess --type execute --verbose=4`
  - `xcrun stapler validate`
- Keep publish gated on those checks passing.

6. Update release docs to match the new release contract.
- Replace unsigned-release guidance in `apps/desktop/README.md` and `DEPLOYMENT_RUNBOOK.md` with the required Apple setup, GitHub secrets, and release verification flow.

## Apple Prerequisites
- Apple Developer Program membership.
- A `Developer ID Application` certificate exported as a `.p12` with private key.
- The certificate identity string for `CSC_NAME`.
- The `.p12` password for `CSC_KEY_PASSWORD`.
- Apple ID with 2FA enabled.
- An app-specific password for notarization.
- The correct Apple Team ID.
