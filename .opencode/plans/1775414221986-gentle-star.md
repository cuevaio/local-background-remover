# Fix A/B Testing And Vercel Flags In `apps/web`

## Goal
- Make Vercel Flags-backed experiments work in both local and deployed environments.
- Stop the silent all-`control` fallback.
- Restore local Toolbar / Flags Explorer support.
- Preserve `exp` across internal navigation so attribution survives to checkout and thank-you flows.

## Root Cause Summary
1. `apps/web/src/lib/experiments/flags.ts` silently falls back to `decide: () => "control"` whenever `process.env.FLAGS` is missing, so misconfiguration looks like a valid control bucket.
2. Remote Vercel Flags state may not match the six code-defined flags, so even a linked project can still behave as if experiments are disabled.
3. Local Toolbar integration is missing from `next.config.mjs` and `src/app/layout.tsx`, so the debugging surface for flags is incomplete.
4. `exp` is computed on experiment pages but dropped on many internal links, which breaks attribution and makes experiment results appear unreliable.

## Implementation Plan
1. Enforce required Vercel Flags envs.
   - Add `FLAGS` and `FLAGS_SECRET` to `apps/web/src/env.ts`.
   - Treat missing values as config errors instead of implicit fallbacks.

2. Remove the silent control fallback in `apps/web/src/lib/experiments/flags.ts`.
   - Replace `getAdapter()` so it requires `env.FLAGS` and throws a clear error if the flag connection string is missing.
   - Keep the existing six flag declarations and variant options unchanged.
   - Keep flag definitions as the source of truth for discovery and page evaluation.

3. Add local Vercel Toolbar support.
   - Add `@vercel/toolbar` to `apps/web/package.json`.
   - Wrap `apps/web/next.config.mjs` with the Toolbar plugin.
   - Render `<VercelToolbar />` from `apps/web/src/app/layout.tsx` so local debugging and overrides work.

4. Preserve `exp` across internal navigation with one reusable link abstraction.
   - Keep `withExpParam()` in `apps/web/src/lib/experiments/attribution.ts` as the canonical URL helper.
   - Add `apps/web/src/components/experiments/ExpLink.tsx` as a small client component that reads the current `exp` from `useSearchParams()` and appends it to internal app links.
   - Avoid changing external links.

5. Replace raw internal links on the main conversion path.
   - Update `src/app/layout.tsx`, `src/app/page.tsx`, `src/app/pricing/page.tsx`, `src/app/downloads/page.tsx`, `src/components/marketing/StickyCta.tsx`, and `src/components/marketing/SiteFooter.tsx` to use `ExpLink` where links should preserve attribution.
   - Prioritize links that move users between home, pricing, downloads, docs, compare, and thank-you related paths.

6. Reconcile remote Vercel Flags state with code using the CLI.
   - Run all `vercel flags` commands from `apps/web`, which is already linked to the correct Vercel project.
   - Inspect existing flags, create missing ones, and update mismatched variants/descriptions so remote config matches `src/lib/experiments/flags.ts` exactly.
   - Refresh local env with `vercel env pull .env.local --yes` after reconciliation.
   - Ensure `FLAGS_SECRET` exists in development, preview, and production.

## Vercel CLI Repair Sequence
Run from `apps/web`:

```bash
vercel flags list
vercel flags inspect home-hero-headline
vercel flags inspect home-primary-cta
vercel flags inspect pricing-hero-copy
vercel flags inspect pricing-plan-cta
vercel flags inspect sticky-cta-copy
vercel flags inspect downloads-hero-copy
```

Create any missing string flags so they match code:

```bash
vercel flags create home-hero-headline --kind string --description "Homepage hero headline copy variant" --variant control="Control" --variant outcome="Outcome" --variant privacy="Privacy"
vercel flags create home-primary-cta --kind string --description "Homepage primary CTA label variant" --variant control="Control" --variant download_free="Download Free" --variant start_downloads="Start Downloads"
vercel flags create pricing-hero-copy --kind string --description "Pricing hero title and subtitle variant" --variant control="Control" --variant one_payment="One Payment" --variant upgrade_later="Upgrade Later"
vercel flags create pricing-plan-cta --kind string --description "Pricing plan card CTA copy variant" --variant control="Control" --variant get_access="Get Access" --variant unlock="Unlock"
vercel flags create sticky-cta-copy --kind string --description "Sticky CTA labels variant" --variant control="Control" --variant compare_install="Compare/Install" --variant buy_get="Buy/Get"
vercel flags create downloads-hero-copy --kind string --description "Downloads hero copy variant" --variant control="Control" --variant install_minutes="Install Minutes" --variant download_then_buy="Download Then Buy"
```

Validate env state and refresh local env:

```bash
vercel env ls development
vercel env ls preview
vercel env ls production
vercel env pull .env.local --yes
```

If `FLAGS_SECRET` is missing, generate and add it:

```bash
node -e "console.log(crypto.randomBytes(32).toString('base64url'))"
vercel env add FLAGS_SECRET development
vercel env add FLAGS_SECRET preview
vercel env add FLAGS_SECRET production
```

Use temporary non-control assignments to smoke test behavior:

```bash
vercel flags set home-hero-headline --environment development --variant outcome --message "Smoke test local variant"
vercel flags set pricing-plan-cta --environment development --variant unlock --message "Smoke test local variant"
vercel flags set downloads-hero-copy --environment preview --variant install_minutes --message "Smoke test preview variant"
```

2. Local Toolbar / Explorer.

3. Local variant behavior.

   - Confirm `/`, `/pricing`, and `/downloads` visibly change copy away from control.

4. Attribution continuity.
   - Start from a URL with `?exp=...`.
   - Click through header, footer, page CTAs, and sticky CTA.
   - Confirm `exp` persists across internal navigation and reaches pricing/downloads/thank-you flows.

5. Checkout handoff.
   - Start checkout from pricing.
   - Confirm `/api/checkout` receives `exp` and that the success URL keeps `exp`.

6. Preview / production smoke test.
   - Force a non-control preview variant and confirm copy changes on the preview deployment.
   - Confirm the deployed `/.well-known/vercel/flags` endpoint responds.
   - After preview validation, briefly verify a production variant if needed and restore the intended remote state.

## Notes
- Recommended policy is to hard-fail everywhere when `FLAGS` is missing.
- Keep the code-defined flag keys, descriptions, and variants as the single source of truth.
- Start with URL-based attribution preservation only; do not add cookie/session persistence unless the URL-based fix proves insufficient.
