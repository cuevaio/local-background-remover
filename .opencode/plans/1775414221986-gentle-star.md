# Expand Marketing Experiments, Simplify Copy, And Improve SEO In `apps/web`

## Goal
- Make the marketing funnel pricing-first instead of download-first.
- Keep `/downloads` public, but demote it from the main pre-purchase journey.
- Rewrite public-page copy for non-technical buyers while still presenting the CLI as an optional advanced tool for developers and coding agents.
- Launch broader A/B coverage on high-intent pages.
- Improve SEO across all public marketing pages, especially around local/offline background removal.

## Confirmed Decisions
- `/downloads` stays public, but is de-emphasized.
- SEO should prioritize local/offline remover intent first.
- Launch many tests at once, but keep them interpretable.
- No Search Console input is available, so keyword targeting should use best judgement from page intent and current site structure.

## Recommended Approach
1. Expand the experiment model without duplicating slots.
   - Reuse existing route-level experiment plumbing in `src/lib/experiments/types.ts` and `src/lib/experiments/flags.ts`.
   - Prefer replacing or expanding existing slot flags instead of adding near-duplicate ones.
   - Add route evaluators only for pages that should actually run tests: home, pricing, downloads, compare hub/detail, docs, and gallery.
   - Keep low-traffic trust pages (`about`, `contact`, `customers`, `privacy`, `faq`, `thank-you`) as content/SEO improvements, not experiment-heavy surfaces.

2. Launch a broad but simple test set on high-intent pages.
   - Use mostly 2-way tests; reserve 3-way tests only for hero framing where the current code already uses multi-variant copy.
   - Recommended flag set:
   - `home-hero-story`: `control | mac_app | product_photos`
   - `home-primary-cta`: replace the current download-led variants with pricing-led variants such as `pricing_first | see_plans | buy_once`
   - `home-cli-emphasis`: `control | advanced_tool`
   - `pricing-hero-copy`: simplify around one-time Mac app value and optional CLI
   - `pricing-plan-framing`: `control | buyer_labels`
   - `pricing-featured-plan`: `bundle_featured | app_featured`
   - `downloads-page-role`: `control | support_page`
   - `compare-primary-cta`: `control | pricing_first`
   - `docs-hero-framing`: `control | agents_and_batches`
   - `gallery-hero-intent`: `control | product_photos`
   - `sticky-cta-copy`: replace download-first secondary language with pricing-first/site-support language
   - After code changes, create/update the matching Vercel flags via CLI/API and activate production splits using the existing `user.id` bucketing.

3. Shift shared navigation and CTA hierarchy to a pricing-first funnel.
   - In `src/app/layout.tsx`, keep pricing as the strongest global CTA.
   - Reframe `Docs` as clearly developer-oriented, for example `CLI Docs` or `Developer Docs`.
   - In `src/components/marketing/SiteFooter.tsx`, make pricing the clear primary footer CTA and keep downloads as secondary support.
   - Remove footer links to `/thank-you` and `/install`, and replace them with crawlable public pages such as `gallery`, `faq`, `compare`, or `pricing`.

4. Rewrite homepage and pricing for non-technical buyers first.
   - `src/app/page.tsx`: lead with outcomes like product photos, portraits, privacy, Mac app, and one-time purchase; move CLI lower and label it as an advanced option for developers/coding agents.
   - `src/components/marketing/WorkflowComparison.tsx`: make the app the default recommendation for most people, with CLI framed as optional automation.
   - `src/components/marketing/AutomationChats.tsx`: keep the coding-agent story, but label it clearly as advanced.
   - `src/app/pricing/page.tsx` and `src/app/pricing/PricingClient.tsx`: replace jargon like `surfaces`, `workflow`, and `automation pipeline` with plain buyer language such as `Mac app`, `command-line tool`, `best for most people`, and `best for scripts and coding agents`.
   - Rewrite pricing plan bullets to explain who each plan is for, not just how it is implemented.

5. Reposition downloads and docs as support pages instead of the default purchase path.
   - `src/app/downloads/page.tsx`: keep the page public and indexable, but make it clear that pricing is the main decision page.
   - Make the desktop app the default install story for non-technical users and move the CLI tab/section into an explicitly advanced position.
   - Rewrite bundle/key language in plain English.
   - `src/app/docs/page.tsx`: keep it technically useful, but clearly state it is for developers, scripts, and coding agents; point most buyers back to pricing/app language.

6. Rewrite compare, gallery, and trust pages so they support search and conversion better.
   - `src/app/compare/page.tsx` and `src/app/compare/[slug]/page.tsx`: make pricing the primary CTA, reduce browser/cloud/dev jargon, and add stronger buyer-language comparisons.
   - `src/content/compare-pages.ts`: add more unique competitor-specific copy, more differentiated FAQs, and stronger source/freshness treatment instead of relying on the same template language everywhere.
   - `src/app/gallery/page.tsx`: position it for product-photo and marketplace use cases, and make pricing the primary CTA.
   - `src/app/faq/page.tsx`: front-load practical buyer questions like which plan to choose, whether the app works offline, and whether CLI is optional.
   - `src/app/about/page.tsx`, `src/app/contact/page.tsx`, `src/app/customers/page.tsx`, and `src/app/privacy/page.tsx`: replace builder/shipper language with broader customer language such as sellers, creators, photographers, designers, marketers, and small teams.
   - `src/app/thank-you/page.tsx`: tailor next steps more clearly by purchase type, with less pre-purchase jargon leaking into the post-purchase page.

7. Run the SEO pass after the copy and funnel structure settle.
   - `src/app/layout.tsx`: add global `Organization` and `WebSite` JSON-LD, and update default metadata so the site leads with local/offline Mac background removal instead of app+CLI parity.
   - `src/lib/seo.ts`: tighten shared title/description/OG defaults around local, private, offline removal and buyer-friendly wording.
   - `src/app/page.tsx`: retarget metadata and schema toward local/offline background remover, background remover for Mac, and product photo cleanup.
   - `src/app/pricing/page.tsx` and `src/app/downloads/page.tsx`: add stronger commercial schema such as `SoftwareApplication` plus `Offer`-style pricing/install context.
   - `src/app/gallery/page.tsx`, `src/app/about/page.tsx`, `src/app/contact/page.tsx`, `src/app/customers/page.tsx`, and `src/app/privacy/page.tsx`: replace generic titles/descriptions with query-oriented metadata.
   - `src/app/sitemap.ts`: add `/gallery` and `lastModified` values, especially for compare pages using `lastReviewedAt`.
   - `src/app/robots.ts`: keep `/thank-you` and `/install` blocked, but align internal linking so blocked/noindex pages are not treated like core resources.

## Verification
1. Content and funnel review
   - Read every public marketing route and confirm the first screen is understandable to a non-technical buyer.
   - Confirm primary CTAs now usually route to `/pricing`, with `/downloads` kept secondary.
   - Confirm the CLI remains visible but is consistently framed as advanced or developer-oriented.

2. Experiment verification
   - Run `bun run build` in `apps/web`.
   - Confirm every newly tested route evaluates its flags, emits `FlagValues`, and records exposures with `ExperimentExposureTracker`.
   - Confirm `exp` still survives navigation through `ExpLink` and reaches checkout/thank-you.
   - Use Vercel CLI/API to inspect each new flag and confirm production splits are active on `user.id`.

3. Live behavior verification
   - Confirm live pages set the stable visitor cookie used for flag bucketing.
   - Verify at least one non-control variant can be forced locally/preview for each new experiment surface.
   - Spot-check production responses across home, pricing, compare, docs, gallery, and downloads to ensure no page falls back to a broken or overly technical state.

4. SEO verification
   - Confirm every public page has a unique title, description, canonical, and strong H1.
   - Confirm `/gallery` is present in the sitemap.
   - Confirm sitemap entries include `lastModified` where applicable.
   - Confirm footer/internal links no longer point to `/thank-you` or `/install`.
   - Confirm JSON-LD renders for home, pricing, downloads, docs, FAQ, compare hub/detail, and global layout schema.

5. Checks to run
   - `cd apps/web && bun run test`
   - `cd apps/web && bun run build`

## Delivery Notes
- Launch many tests at once, but keep most of them to one page-level narrative decision plus one CTA decision so results stay readable.
- Treat low-traffic trust pages as copy and SEO improvements, not as separate optimization experiments.
- Use the existing Vercel Flags + visitor-cookie setup as the single bucketing system for all new tests.
