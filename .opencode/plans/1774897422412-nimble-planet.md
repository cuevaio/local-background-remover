# Desktop App: Shippable macOS MVP Plan

## Goal
Deliver a shippable macOS desktop MVP that works outside the monorepo dev environment, preserves dual-surface licensing (`desktop` + `cli`), supports real save/export behavior, and provides clear actionable error handling.

## Scope Decisions
- Target: distributable macOS app artifacts (`.dmg`/`.zip`) with bundled `rmbg` runtime.
- Keep existing Electron architecture (main + preload + renderer) and persistent worker model.
- Keep licensing source of truth in CLI; desktop remains orchestrator.
- Non-goals for this milestone: auto-update, telemetry platform, broad UI redesign.

## Phase 1 - Runtime Packaging and Resolution
1. Add a desktop runtime resolver module to locate CLI in this order:
   - `RMBG_DESKTOP_CLI_PATH` (explicit override for QA/debug)
   - packaged runtime under `process.resourcesPath`
   - dev fallback: `apps/rmbg/.venv/bin/rmbg`
   - final dev fallback: `uv run --project apps/rmbg rmbg`
2. Add a prepare script that stages the packaged CLI runtime into desktop build resources.
3. Update main process subprocess bootstrapping to use resolver output for one-shot calls and worker startup.

## Phase 2 - License Enforcement Consistency (Worker + CLI)
1. Extend worker invocation contract to carry license context (surface/required surfaces/api base/license file as needed).
2. Enforce license checks inside worker actions (`ensure_model`, `remove`) using existing license-manager helpers, not only Electron prechecks.
3. Preserve existing offline-after-activation and grace behavior by reusing current `license_manager` flow.
4. Align `--surface`/`--require-surface` semantics across desktop docs and runtime behavior.

## Phase 3 - Error Contract Hardening
1. Add desktop-side error normalization for CLI and worker failures:
   - parse JSON failures when present
   - preserve exit code context
   - map to stable categories (`license`, `model`, `input`, `inference`, `output`, `runtime`)
2. Return renderer-safe structured errors through IPC, with clear user-facing messages.
3. Keep detailed diagnostics available in main process for troubleshooting.

## Phase 4 - Complete Save/Export UX
1. Implement real Save flow in desktop:
   - IPC handler opens native save dialog
   - copy/move processed output to selected path
   - handle cancel and overwrite paths safely
2. Update renderer card actions so `Save Result` performs actual file output, not status-only text.
3. Keep current before/after compare workflow and explicit states (`idle`, `processing`, `done`, `failed`).

## Phase 5 - Packaging Pipeline
1. Add Electron packaging configuration (electron-builder) for macOS artifacts.
2. Wire scripts to produce desktop distributables with bundled CLI runtime.
3. Support unsigned local QA packaging first; include env-driven signing/notarization hooks for release builds.
4. Document build/release steps and required environment variables in desktop README.

## Phase 6 - Replace Placeholder Quality Gates
1. Replace desktop placeholder lint/test scripts with meaningful checks:
   - lint: at minimum TypeScript no-emit typecheck
   - test: lightweight unit tests for runtime resolver and error parser
2. Keep tests focused and fast; avoid heavyweight UI automation in this MVP.

## Verification Plan
### Targeted CLI/worker checks
- `cd apps/rmbg && uv run --group dev ruff check rmbg_cli tests`
- `cd apps/rmbg && uv run python -m py_compile rmbg_cli/cli.py`
- `cd apps/rmbg && uv run --group dev pytest tests/test_cli_license.py -q`
- `cd apps/rmbg && uv run --group dev pytest tests/test_license_manager.py -q`
- `cd apps/rmbg && uv run --group dev pytest tests/test_worker_server.py -q`

### Desktop checks
- `cd apps/desktop && bun run build`
- `cd apps/desktop && bun run lint`
- `cd apps/desktop && bun run test`

### Packaging checks
- `cd apps/rmbg && bun run package:mac`
- `cd apps/desktop && bun run dist:mac`
- Confirm packaged app includes executable `rmbg` runtime under app resources.

### End-to-end manual smoke
1. Launch packaged app on macOS (clean user profile preferred).
2. Verify dual activation flow (`desktop` key + `cli` key).
3. Verify `Ensure Model` is blocked before activation and succeeds after activation.
4. Process at least one image and confirm output preview updates.
5. Use `Save Result` to custom destination and verify file exists and opens.
6. Disable network after activation and verify processing still works within active/grace window.
7. Verify clear user-facing errors for license, model bootstrap, input read, inference, and output write failures.
