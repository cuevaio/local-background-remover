# SEO Blog Section Plan

## Goal
- Add a first-class `/blog` section in `apps/web` to publish 10 static SEO landing posts that improve organic coverage for background-removal use cases and drive readers into pricing, downloads, docs, and compare pages.

## Recommended Approach
- Keep the existing `/blog` and `/blog/[slug]` App Router structure and migrate blog content out of TypeScript into plain `.md` files.
- Use YAML frontmatter for SEO and routing metadata, with normal markdown for the article body.
- Do not introduce MDX unless posts need embedded React components later; plain markdown is the simpler fit for this repo and this SEO content.
- Preserve sitemap, metadata, JSON-LD, and nav/footer discoverability.

## Files To Add
- `apps/web/content/blog/*.md`
  - Store one post per markdown file.
  - Use YAML frontmatter for fields like: `slug`, `title`, `description`, `excerpt`, `eyebrow`, `publishedAt`, `updatedAt`, `targetKeyword`, `heroHighlights`, CTA links, related slugs, and FAQs.
- `apps/web/src/content/blog.ts`
  - Add a small build-time loader that reads markdown files, parses frontmatter, and returns typed blog post data plus rendered markdown content.
  - Export helpers like `getAllBlogPosts()`, `getBlogPostBySlug(slug)`, and `getBlogSlugs()`.
- `apps/web/src/app/blog/page.tsx`
  - Render the blog hub page.
  - List all posts from the markdown loader with title, excerpt, keyword/use-case framing, updated date, and CTA to the detail page.
  - Export `metadata` via `buildPageMetadata({ title, description, path: "/blog" })`.
  - Add `BreadcrumbList` plus `ItemList` or `CollectionPage` JSON-LD.
- `apps/web/src/app/blog/[slug]/page.tsx`
  - Implement `generateStaticParams()` from markdown slugs.
  - Implement `generateMetadata()` using frontmatter title/description and canonical path.
  - Render each post using frontmatter plus markdown body, with CTA block, related posts, and visible FAQs when provided.
  - Return `notFound()` for invalid slugs.
  - Add `BreadcrumbList`, `BlogPosting`, and conditional `FAQPage` JSON-LD.

## Files To Modify
- `apps/web/package.json`
  - Add the minimal markdown parsing dependencies needed for frontmatter and markdown rendering.
- `apps/web/src/app/layout.tsx`
  - Add `Blog` to the header navigation using `ExpLink`.
  - Keep visibility aligned with existing marketing links on desktop and mobile breakpoints.
- `apps/web/src/components/marketing/SiteFooter.tsx`
  - Add `/blog` to the footer so the section is crawlable and user-discoverable from every page.
  - Keep the footer column count stable by swapping or rebalancing links if needed.
- `apps/web/src/app/sitemap.ts`
  - Import blog data from the markdown loader.
  - Add `/blog` plus all `/blog/[slug]` URLs.
  - Use `updatedAt` for per-post `lastModified`.
- `apps/web/src/app/robots.ts`
  - Review whether the file uses explicit allow rules; if so, include `/blog` for consistency.

## Content Plan
- Create 10 SEO landing posts focused on product-adjacent search intent.
- Target topics such as:
  - remove background on Mac
  - best background remover for Mac
  - offline background remover
  - private background remover
  - batch background removal on Mac
  - background remover for product photos
  - remove background without Photoshop
  - local remove.bg alternative
  - command line background removal
  - background remover for ecommerce sellers
- Each post should have:
  - a unique title and meta description
  - a clear target keyword
  - practical markdown body content tied to the product's differentiators
  - internal links to relevant pages like `/pricing`, `/downloads`, `/docs`, `/compare`, and other blog posts
  - at least one strong CTA

## SEO Details
- Use `buildPageMetadata()` everywhere for canonical and social consistency.
- Keep slug paths identical across metadata, JSON-LD, sitemap entries, and internal links.
- Use `BlogPosting` schema on post pages and only emit `FAQPage` schema when the FAQ is visibly rendered.
- Prefer static generation for the entire section so pages are indexable and predictable.
- Cross-link related posts and relevant commercial pages to improve crawl depth and conversion paths.

## Implementation Notes
- Match existing App Router and SEO conventions, but move post authoring into filesystem markdown for easier edits.
- Prefer plain `.md` with frontmatter over MDX because the site does not currently use MDX and these posts are content-first rather than component-first.
- Keep the loader small and build-time only; avoid CMS/contentlayer-style infrastructure unless the blog grows significantly.
- If the compare pages already establish visual section patterns worth reusing, borrow those patterns directly rather than inventing a separate blog design system.
