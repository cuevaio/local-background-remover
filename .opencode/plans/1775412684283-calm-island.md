# Hide Internal Surface Flags From CLI Help

## Goal
- Keep `--surface` and `--require-surface` fully functional for desktop/internal callers.
- Hide those flags from user-facing CLI help and documentation so normal CLI users do not see or need them.
- Preserve current runtime and parsing behavior.

## Root Cause
- `apps/rmbg/rmbg_cli/cli.py` currently exposes `--surface` and `--require-surface` in `--help` across several commands.
- The root parser epilog and `apps/rmbg/README.md` also document those flags directly.
- That makes an internal entitlement detail visible to end users even though the desktop app should manage it automatically.

## Recommended Changes
1. In `apps/rmbg/rmbg_cli/cli.py`, keep the existing `--surface` and `--require-surface` arguments, defaults, and choices unchanged.
2. Set their `help` to `argparse.SUPPRESS` anywhere they appear so parsing still works but help output hides them.
3. Remove `--surface` usage from the root parser epilog examples.
4. Apply the same `help=argparse.SUPPRESS` change to the internal parser in `apps/rmbg/rmbg_cli/worker_server.py`.
5. Update `apps/rmbg/README.md` command examples to remove surface-related flags from user-facing docs.
6. Remove or rewrite any doc text that tells CLI users to think about surface activations.

## Files To Modify
- `apps/rmbg/rmbg_cli/cli.py`
- `apps/rmbg/rmbg_cli/worker_server.py`
- `apps/rmbg/README.md`
- `apps/rmbg/tests/test_cli_license.py`

## Test Plan
1. Keep the existing parser tests that explicitly pass `--surface` / `--require-surface` so hidden flags are still proven to parse.
2. Add minimal help-output regression tests in `apps/rmbg/tests/test_cli_license.py`:
   - assert root help does not contain `--surface`
   - assert a representative subcommand help does not contain `--surface` or `--require-surface`
3. No broader behavior changes should be needed because runtime logic stays untouched.

## Verification
- `cd apps/rmbg && uv run python -m py_compile rmbg_cli/cli.py rmbg_cli/worker_server.py`
- `cd apps/rmbg && uv run --group dev pytest tests/test_cli_license.py -q`
- Manual smoke checks:
  - `cd apps/rmbg && uv run rmbg --help`
  - `cd apps/rmbg && uv run rmbg remove --help`
  - `cd apps/rmbg && uv run python -m rmbg_cli.worker_server --help`

## Risks And Assumptions
- `argparse.SUPPRESS` should hide the options from help output without affecting parsing, defaults, or choices.
- This change intentionally hides only the UI/docs surface, not the internal behavior, so desktop flows should continue to work unchanged.
