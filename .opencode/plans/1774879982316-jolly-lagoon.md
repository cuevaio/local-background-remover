# Plan: URL Image Input Support for `rmbg remove`

## Goal
- Allow users to pass an image URL to `--input` (not just local file paths), download it safely, and run background removal.
- Keep behavior consistent for both CLI and worker-based remove flows by implementing this in shared image I/O.

## Confirmed Product Decisions
- **Scope:** shared loader support so both direct CLI remove and worker remove paths accept URLs.
- **Safety defaults:** allow only `http`/`https`, timeout `20s`, max download size `20MB`, follow redirects, reject non-image responses, and return actionable errors.
- **Default output (URL input, no `--output`):** derive from URL basename (sanitized), fallback to `image_rmbg.png` in current working directory.

## Implementation Plan
1. **Extend shared image loader in `apps/rmbg/rmbg_cli/io.py`**
   - Add URL detection helper (e.g. `is_http_url`).
   - Add URL download helper using `urllib` with strict limits:
     - scheme allowlist (`http`, `https`)
     - request timeout `20s`
     - content-type check for `image/*`
     - content-length pre-check when present
     - streamed read cap at `20MB`
   - Decode downloaded bytes with Pillow and keep existing orientation + RGB conversion (`ImageOps.exif_transpose(...).convert("RGB")`).
   - Keep local file loading logic unchanged for non-URL inputs.

2. **Make default output naming URL-aware in `apps/rmbg/rmbg_cli/io.py`**
   - Update `default_output_path(input_ref)`:
     - local path behavior stays unchanged
     - URL behavior derives stem from URL path basename, sanitizes filename-safe stem, appends `_rmbg.png`, writes to cwd
     - fallback filename: `image_rmbg.png`

3. **Adjust CLI success payload path handling in `apps/rmbg/rmbg_cli/cli.py`**
   - In `cmd_remove`, keep URL input string as-is in success payload (`input_path`) instead of applying `Path(...).resolve()` to URL strings.
   - Preserve existing exit-code contract, especially input-read failures returning code `2`.

4. **Worker compatibility (shared path)**
   - No dedicated worker code change required if `load_image` is URL-aware, because worker remove already calls shared loader.
   - Verify worker error surfacing remains clear for URL fetch failures.

5. **Add/expand tests**
   - **Update `apps/rmbg/tests/test_io.py`** for URL behavior:
     - URL success path
     - unsupported scheme rejection
     - non-image content-type rejection
     - oversize response rejection (header and streaming)
     - URL output-name derivation + fallback behavior
   - **Add `apps/rmbg/tests/test_cli_remove.py`**:
     - URL read failure returns CLI exit code `2`
     - success payload keeps raw URL in `input_path`
     - URL default output path behavior when `--output` omitted
   - **Add `apps/rmbg/tests/test_worker_server.py`**:
     - remove action with URL input uses shared loader behavior
     - URL loader failures are surfaced in worker error response
