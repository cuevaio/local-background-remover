## Goal

Update the macOS desktop app icon so packaged builds use the same logo as the web app, with `apps/web/src/app/icon.svg` as the source of truth.

## Findings

- The desktop app currently does not define a custom macOS icon in repo config.
- `apps/desktop/electron-builder.yml` uses the default build resources directory (`build/`) and currently has no `mac.icon` entry.
- Existing packaged output shows Electron's default icon (`electron.icns`) is being bundled.
- The user-selected canonical web logo asset is `apps/web/src/app/icon.svg`.

## Recommended Approach

1. Keep `apps/web/src/app/icon.svg` as the single source of truth for the icon artwork.
2. Generate a macOS `.icns` file from that SVG and place it at `apps/desktop/build/icon.icns`.
3. Add a small helper script in `apps/desktop/scripts/` to regenerate `icon.icns` from the web SVG when branding changes.
4. Add a package script in `apps/desktop/package.json` to run that generator.
5. Do not change `apps/desktop/electron-builder.yml` initially.
6. Verify that `electron-builder` picks up `build/icon.icns` automatically during `bun run dist:mac`.
7. Only add an explicit `mac.icon` entry in `electron-builder.yml` if packaging verification shows the default lookup is insufficient.

## Files To Modify

- `apps/desktop/build/icon.icns`
- `apps/desktop/scripts/generate-mac-icon.mjs`
- `apps/desktop/package.json`
- Optional only if needed after verification: `apps/desktop/electron-builder.yml`

## Implementation Notes

- The generator should read `apps/web/src/app/icon.svg` and create the standard macOS iconset sizes required by `iconutil`.
- Generate on macOS using `iconutil -c icns` and clean up any temporary `.iconset` directory afterward.
- Commit the generated `apps/desktop/build/icon.icns` so local packaging does not depend on regenerating the asset every time.
- The web icon includes thin strokes and a border; small-size outputs should be visually checked for clarity.

## Verification

1. Run the icon generation script and confirm `apps/desktop/build/icon.icns` is created.
2. Run `bun run dist:mac` in `apps/desktop`.
3. Confirm packaging succeeds and writes output to `apps/desktop/dist-packages/`.
4. Open the produced `.app` or mounted `.dmg` and verify the icon in Finder.
5. Launch the built app and verify the icon in the Dock/app switcher.
6. Inspect the packaged bundle metadata/resources if needed to confirm the default Electron icon is no longer being used.

## Notes

- `apps/desktop/scripts/dist-mac.mjs` already supports unsigned local builds when signing env vars are absent, so local verification should be feasible.
- If macOS icon cache makes results ambiguous, verify using a newly built artifact rather than an older installed copy.
