## Implementation Plan

### 1) Add typed compare content source
Create a single source-of-truth for compare pages and SEO fields.

- Create `apps/web/src/content/compare-pages.ts`
- Define typed shape for each page entry (recommended fields):
  - `slug`, `competitorName`, `competitorUrl`
  - `seoTitle`, `seoDescription`
  - `hero` copy + CTA labels/links
  - `comparisonRows` (table-ready rows)
  - `faq` (for on-page FAQ + JSON-LD)
  - `sources` (public URLs used for claims)
  - `lastReviewedAt` and `disclaimer`
- Export helpers:
  - `COMPARE_PAGES`
  - `COMPARE_SLUGS`
  - `getComparePageBySlug(slug)`

Why: this matches current in-repo typed content patterns, avoids CMS overhead, and scales to many pages.

### 2) Build compare hub and dynamic route
- Create `apps/web/src/app/compare/page.tsx` (hub/index page listing all 12 compare pages).
- Create `apps/web/src/app/compare/[slug]/page.tsx`.
- In `[slug]/page.tsx`:
  - Use `generateStaticParams()` from `COMPARE_SLUGS` for static generation.
  - Use `notFound()` for unknown slugs.
  - Render a consistent compare template (hero, table, proof/FAQ, CTA).

### 3) Reuse existing marketing UI + minimal new compare UI
Reuse existing pieces first, only add focused compare components when needed.

- Reuse:
  - `apps/web/src/components/marketing/StickyCta.tsx`
  - `apps/web/src/components/marketing/ProofStrip.tsx`
  - `apps/web/src/components/marketing/SiteFooter.tsx`
  - `apps/web/src/components/ui/table.tsx`
  - `apps/web/src/components/ui/accordion.tsx`
- If needed, add small compare-specific components:
  - `apps/web/src/components/marketing/CompareHero.tsx`
  - `apps/web/src/components/marketing/CompareMatrix.tsx`
  - `apps/web/src/components/marketing/CompareFaq.tsx`

### 4) SEO metadata + structured data per compare page
Follow the current SEO architecture in `apps/web/src/lib/seo.ts`.

- In `apps/web/src/app/compare/[slug]/page.tsx`, add `generateMetadata()` and call `buildPageMetadata(...)` with slug-specific values.
- Add JSON-LD scripts using `serializeJsonLd(...)`:
  - `BreadcrumbList` for crawl clarity
  - `FAQPage` only when FAQ content is present
- Keep claim language factual/non-superlative and include a visible "last reviewed" + trademark disclaimer block.

### 5) Sitemap + internal linking for crawlability
- Update `apps/web/src/app/sitemap.ts` to include:
  - `/compare`
  - all `/compare/<slug>` routes from `COMPARE_SLUGS`
- Update compare links in `apps/web/src/components/marketing/SiteFooter.tsx` (replace `#` placeholders with real compare routes).
- Add at least one crawl path from primary nav or homepage:
  - update `apps/web/src/app/layout.tsx` and/or `apps/web/src/app/page.tsx` with a "Compare" link.

### 6) Content quality safeguards
- Every compare entry must include `sources` and `lastReviewedAt`.
- Avoid unverifiable claims ("best", "#1", "most accurate") unless cited.
- Keep comparisons neutral: "good fit when..." vs "better for..." framing.
