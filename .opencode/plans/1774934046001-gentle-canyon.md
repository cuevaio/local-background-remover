# Plan: App-only desktop license can use CLI runtime through the app only

## Goal
- Let an `app` purchaser activate only in the desktop app and still process images successfully.
- Ensure the desktop app installs or updates the CLI/runtime on first launch so the app installer can stay light.
- Allow the runtime to exist globally on disk, but prevent normal direct terminal use when the user only owns the app.

## Recommended Approach
1. Keep the current web entitlement model.
- Do not change Polar product mapping, benefit checks, or offline token shape.
- `desktop` stays a desktop-only surface and `cli` stays a CLI-only surface.
- The problem is local trust, not token issuance.

2. Add a desktop-host proof for any command that consumes the `desktop` surface.
- Add a helper in `apps/rmbg/rmbg_cli/license_manager.py` that validates a short-lived desktop session proof from environment variables.
- The desktop main process creates a random nonce plus a short expiry in its app data directory and passes the matching env vars on every desktop-launched `rmbg` process.
- In the CLI, require that proof whenever a command uses the `desktop` surface:
  - `license activate --surface desktop`
  - `license refresh --surface desktop`
  - `model ensure --surface desktop`
  - `remove --surface desktop`
  - `worker --surface desktop`
- If the proof is missing or invalid, fail with exit code `6` and a message that says the runtime is licensed for the desktop app only and that terminal use needs a CLI license.
- Keep `license status` readable so diagnostics still work.

3. Bootstrap the runtime on first launch before activation or processing.
- The desktop app currently activates by calling the CLI, so the runtime must exist before the user can finish app activation.
- On startup, or before the first activation attempt, check whether the expected `rmbg` version is installed.
- If it is missing or out of date, run the existing public install script from the desktop app with `RMBG_VERSION` pinned to the desktop app version.
- This keeps the installer light while reusing the current release-download flow.

4. Teach the desktop app to prefer the script-installed runtime.
- Update the packaged-app resolver so it no longer assumes the runtime must be bundled inside the app package.
- Prefer this order in packaged builds:
  - explicit `RMBG_DESKTOP_CLI_PATH`
  - first-launch installed runtime
  - existing bundled runtime as a temporary rollout fallback
- Keep the existing dev resolution path (`.venv` then `uv run`) unchanged.

5. Keep direct terminal CLI behavior separate.
- `rmbg remove ...` should continue to default to the `cli` surface and require a CLI license.
- `rmbg remove --surface desktop ...` should no longer work from Terminal for an app-only customer because it will lack the desktop-host proof.
- This gives the app a normal-use-only path to the runtime without turning the desktop entitlement into a public CLI entitlement.

## Files To Modify
- `apps/desktop/src/main/main.ts`
  - Create/rotate the desktop session proof.
  - Inject proof env vars into desktop-launched CLI and worker spawns.
  - Ensure runtime bootstrap happens before desktop activation and before processing commands.
- `apps/desktop/src/main/rmbg-runtime.ts`
  - Resolve the first-launch installed runtime path and version.
  - Prefer it in packaged builds while keeping existing dev behavior.
- `apps/desktop/src/preload/preload.ts`
  - Expose runtime bootstrap status or retry hooks if the renderer needs them.
- `apps/desktop/src/renderer/renderer.ts`
  - Show a blocking "Installing runtime..." or retry state before activation/processing when the runtime is missing.
- `apps/rmbg/rmbg_cli/license_manager.py`
  - Add desktop-host proof validation and the shared error message for desktop-only runtime use.
- `apps/rmbg/rmbg_cli/cli.py`
  - Enforce the desktop-host proof before desktop-surface activation, refresh, model ensure, remove, and worker startup.
- `apps/rmbg/rmbg_cli/worker_server.py`
  - Reject a desktop worker that is started without valid desktop-host proof.
- `apps/rmbg/tests/test_license_manager.py`
  - Cover valid proof, missing env, nonce mismatch, and expiry.
- `apps/rmbg/tests/test_cli_license.py`
  - Cover desktop activation/refresh gating.
- `apps/rmbg/tests/test_cli_remove.py`
  - Cover desktop-surface remove failure without proof and unchanged CLI behavior.
- `apps/rmbg/tests/test_worker_server.py`
  - Cover worker rejection without proof.
- `apps/desktop/test/rmbg-runtime.test.cjs`
  - Cover runtime resolution precedence for installed vs bundled/dev runtimes.

## Risks
- This is a product guard, not hard local anti-tamper. A determined user could still inspect local state.
- First-launch bootstrap adds a network dependency and needs clear retry/error UI.
- App and CLI versions must stay pinned together to avoid protocol drift.
- Keeping the bundled runtime as a temporary fallback makes rollout safer but should be removed once the first-launch flow is proven.

## Verification
1. `cd apps/rmbg && uv run python -m py_compile rmbg_cli/cli.py`
2. `cd apps/rmbg && uv run --group dev pytest tests/test_license_manager.py tests/test_cli_license.py tests/test_cli_remove.py tests/test_worker_server.py -q`
3. `cd apps/desktop && bun run test`
4. Packaged app smoke test on a machine with no `rmbg` installed:
- first launch installs runtime
- app key activation succeeds in the app
- model ensure succeeds in the app
- one image processes end to end
5. Terminal smoke test for an app-only buyer:
- `rmbg remove ...` fails with CLI-license-required behavior
- `rmbg remove --surface desktop ...` fails with desktop-app-only behavior
- `rmbg worker --surface desktop` fails the same way
6. CLI-owner smoke test:
- public install still works
- `rmbg license activate --surface cli` works
- terminal `rmbg remove ...` works with a CLI license
