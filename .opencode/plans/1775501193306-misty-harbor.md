# Add More Vercel Analytics Events In `apps/web`

## Goal
- Expand Vercel Analytics beyond experiment exposure and basic checkout milestones.
- Prioritize high-intent CTA click tracking across the main conversion funnel.
- Add supporting instrumentation for checkout failures, command copy actions, and license API usage.
- Keep the rollout observational only: no behavior changes, no secret/high-cardinality payloads.

## Recommended Approach

### 1. Extend the existing analytics wrapper
- Update `apps/web/src/lib/analytics/events.ts` to stay the single client-side entry point for custom analytics.
- Add new client wrappers and shared event-name constants for:
  - `cta_clicked`
  - `command_copied`
  - existing `experiment_exposure`
  - existing `checkout_completed_proxy`
- Add a small server-side companion, likely `apps/web/src/lib/analytics/events.server.ts`, to wrap `@vercel/analytics/server` for:
  - `checkout_started`
  - `checkout_failed`
  - `license_api_used`
- Keep event payloads stable and compact. Do not send license keys, machine hashes, activation IDs, Polar IDs, or freeform raw errors.

### 2. Add a tracked link component for server-rendered pages
- Create a small client component, likely `apps/web/src/components/analytics/TrackedExpLink.tsx`.
- Compose the existing `apps/web/src/components/experiments/ExpLink.tsx` rather than replacing its attribution behavior.
- Use it only for high-intent CTA placements where server components currently cannot attach click handlers directly.
- Recommended `cta_clicked` payload fields:
  - `slot`: stable placement id like `home.hero.primary` or `footer.primary`
  - `label`: rendered CTA text
  - `href`: destination path
  - `page_path`: current pathname
  - `exp`: current experiment token
  - `has_exp`: boolean
  - `kind`: optional for pricing buttons (`app`, `cli`, `both`)

### 3. Instrument high-intent CTA clicks
- Use `TrackedExpLink` in the main high-intent funnel surfaces:
  - `apps/web/src/app/layout.tsx`
    - Track only the strongest persistent conversion CTA: `Buy once`
    - Skip broader nav links for this pass
  - `apps/web/src/components/marketing/StickyCta.tsx`
    - Track primary and secondary sticky CTA buttons
  - `apps/web/src/components/marketing/SiteFooter.tsx`
    - Track `View one-time pricing`
    - Track `Install anytime`
    - Track `Read the indie story` only if included as a deliberate bottom-funnel CTA
  - `apps/web/src/app/page.tsx`
    - Track top hero and other obvious conversion CTAs (pricing/install/gallery/docs where they are presented as CTA buttons)
  - `apps/web/src/app/downloads/page.tsx`
    - Track install-related CTA buttons and key paths to pricing/docs
  - `apps/web/src/app/docs/page.tsx`
    - Track hero/sticky CTA buttons that push users toward install or purchase

### 4. Instrument pricing plan interactions and checkout outcomes
- Update `apps/web/src/app/pricing/PricingClient.tsx`:
  - Fire `cta_clicked` before starting checkout for each plan button
  - Use stable slots like:
    - `pricing.plan.app`
    - `pricing.plan.cli`
    - `pricing.plan.both`
  - Include `kind`, `label`, `page_path`, `exp`, and `has_exp`
- Update `apps/web/src/app/api/checkout/route.ts`:
  - Replace direct `@vercel/analytics/server` calls with the new server wrapper
  - Keep `checkout_started` as the canonical server event after checkout URL creation succeeds
  - Add `checkout_failed` for invalid payloads and upstream checkout creation failures
- Recommended checkout payload fields:
  - `kind`
  - `page: "pricing"`
  - `exp`
  - `has_exp`
  - `reason` for failures using low-cardinality buckets like `invalid_kind`, `polar_error`, `unknown`

### 5. Track command copy actions in docs/download flows
- Update `apps/web/src/components/marketing/CommandBlock.tsx` to emit `command_copied` when clipboard write succeeds.
- Add a new prop like `commandId` so analytics uses stable IDs instead of raw command text.
- Update `CommandBlock` usages in:
  - `apps/web/src/app/docs/page.tsx`
  - `apps/web/src/app/downloads/page.tsx`
- Recommended `command_copied` payload fields:
  - `command_id`
  - `page_path`
  - `exp`
  - `has_exp`
- Suggested IDs based on current docs content:
  - `cli_install`
  - `cli_version`
  - `cli_activate`
  - `cli_status`
  - `cli_model_ensure`
  - `cli_remove`
  - any other explicitly rendered command blocks in docs/downloads

### 6. Track license API usage on the server
- Instrument these routes:
  - `apps/web/src/app/api/license/activate/route.ts`
  - `apps/web/src/app/api/license/refresh/route.ts`
  - `apps/web/src/app/api/license/validate/route.ts`
- Emit a shared `license_api_used` server event from success and failure paths.
- Recommended payload fields:
  - `action`: `activate` | `refresh` | `validate`
  - `surface`: `cli` | `desktop` | `unknown`
  - `ok`: boolean
  - `status_code`
  - `result`: normalized bucket like `granted`, `inactive`, `missing_params`, `wrong_entitlement`, `error`
  - `has_activation_id`: boolean when relevant
- Keep the API responses unchanged.

## Verification

### Local checks
- Run `bun run dev` from `apps/web`.
- Smoke test the main pages:
  - `/`
  - `/pricing`
  - `/downloads`
  - `/docs`
- Confirm no UI or routing regressions after replacing CTA links with tracked versions.
- Confirm `exp` attribution still propagates correctly through internal links.

### Analytics behavior checks
- Click tracked high-intent CTAs and verify `cta_clicked` appears in Vercel Analytics live/debug views.
- Start checkout from each pricing plan and confirm:
  - `cta_clicked` fires on button click
  - `checkout_started` fires on successful checkout creation
- Trigger an invalid checkout request and confirm `checkout_failed` appears with a normalized `reason`.
- Copy commands on docs/downloads pages and confirm `command_copied` appears with stable `command_id` values.
- Exercise license endpoints with controlled invalid payloads and confirm `license_api_used` is emitted without leaking sensitive values.

### Scope guardrails
- Do not instrument every footer or nav link in this pass.
- Do not add authoritative payment completion tracking here unless the scope expands to Polar webhook-based instrumentation.
- Do not alter checkout, license, or navigation behavior beyond analytics hooks.

## Suggested Implementation Order
1. Extend analytics wrappers in `events.ts` and add `events.server.ts`
2. Create `TrackedExpLink`
3. Instrument pricing CTA clicks and checkout route events
4. Instrument sticky/footer/home/docs/downloads high-intent CTAs
5. Instrument `CommandBlock` copy events
6. Instrument license API routes
