# Plan: Deploy + Full CLI Flow (Install -> Activate -> Use)

## Direct Answers
- CLI framework in use: **`argparse`** (`apps/rmbg/rmbg_cli/cli.py`).
- Final binary size: **not lightweight** (Torch-based).
  - expected per-arch `.tar.gz`: roughly **180–350 MB**
  - expected unpacked footprint: roughly **450–900 MB**
- Artifact hosting recommendation:
  - **v1**: keep binaries in **GitHub Releases** (public)
  - do **not** commit binaries to git repo history
  - later scale: move to object storage/CDN if install volume grows.

## Deployment Target (Confirmed)
- Web/API hosting: **Vercel** (`apps/web`)
- Binary hosting: **GitHub Releases**
- Public installer URL: `https://local.backgroundrm.com/install`
- License model: public downloads + runtime activation required + offline-after-activation token policy.

## Goal
- Validate the complete production path:
  1) user installs CLI from `curl -fsSL https://local.backgroundrm.com/install | bash`
  2) user activates key
  3) user runs background removal successfully
  4) runtime remains license-gated and supports offline-after-activation windows.

## Phase 1 - Preflight and Release Hygiene
- Ensure clean release candidate on `main` and pick target tag `vX.Y.Z`.
- Verify local quality gates:
  - `bun install`
  - `bun run lint`
  - `bun run test`
  - `cd apps/rmbg && uv sync --group dev`
  - `cd apps/rmbg && uv run --group dev pytest -q`
- Confirm installer/release contract is consistent:
  - release archive names match installer expectations (`rmbg-vX.Y.Z-darwin-*.tar.gz`)
  - `checksums.txt` is published with each release.

## Phase 2 - Deploy Web API (Vercel)
- Deploy `apps/web` to production on Vercel with `local.backgroundrm.com` domain.
- Configure required production env vars:
  - `POLAR_SERVER=production`
  - `POLAR_ACCESS_TOKEN`, `POLAR_ORGANIZATION_ID`
  - `POLAR_PRODUCT_*` and `POLAR_BENEFIT_*`
  - `LICENSE_SIGNING_PRIVATE_KEY`
- Verify production routes:
  - `POST /api/license/activate`
  - `POST /api/license/refresh`
  - `POST /api/license/validate`
  - `GET /install` returns shell script (`text/x-shellscript`).

## Phase 3 - Publish CLI Binaries (GitHub Releases)
- Create and push tag `vX.Y.Z` to trigger `.github/workflows/release.yml`.
- Confirm release contains:
  - `rmbg-vX.Y.Z-darwin-arm64.tar.gz`
  - `rmbg-vX.Y.Z-darwin-x86_64.tar.gz`
  - `checksums.txt`
- Verify checksums from downloaded release assets.

## Phase 4 - Installer Endpoint Validation
- Validate production installer command:
  - `curl -fsSL https://local.backgroundrm.com/install | bash`
- Confirm installed binary path and executable:
  - `command -v rmbg`
  - `rmbg --version`
- Confirm installer behavior on both macOS arch types (arm64/x86_64).

## Phase 5 - End-to-End Runtime Test (User Journey)

### A) First-time install + activation
- Install with public command.
- Export runtime verification vars if not embedded:
  - `RMBG_LICENSE_API_BASE_URL=https://local.backgroundrm.com`
  - `RMBG_LICENSE_PUBLIC_KEY=<base64-ed25519-public-key>`
- Activate:
  - `rmbg license activate --key YOUR_KEY --surface cli --format json`
- Verify status:
  - `rmbg license status --surface cli --format json`

### B) First-time model bootstrap + inference
- Ensure model:
  - `rmbg model ensure --surface cli --format json`
- Remove background:
  - `rmbg remove --surface cli --input ./butterfly.jpg --output /tmp/out.png --format json`

### C) License gating checks
- Unlicensed machine/cache:
  - `model ensure` and `remove` must fail with license code/message.
- Activated machine:
  - both commands must pass.
- Desktop integration:
  - desktop calls must pass `--surface desktop` for ensure/remove (already expected behavior).

### D) Offline-after-activation checks
- With valid token in active/grace, run offline and confirm success.
- Past grace window, confirm license failure and refresh requirement.

## Rollback Plan
- Web/API rollback: revert to previous Vercel deployment.
- CLI rollback:
  - support pinned install version via `RMBG_VERSION=vPrev` during incident response.
  - publish fixed patch release rather than deleting tags.
- Communicate temporary pinned install command to users if latest release regresses.

## Go-Live Checklist
- `https://local.backgroundrm.com/install` responds correctly.
- Release artifacts + checksums are available for latest tag.
- Fresh-machine install succeeds.
- Activation works for valid key and rejects invalid entitlement.
- Unlicensed runtime commands are blocked.
- Licensed runtime commands complete successfully.
- Offline-after-activation behavior is verified.
