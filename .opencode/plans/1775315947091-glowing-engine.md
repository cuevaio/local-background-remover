# Homepage Slider-First Refresh

## Goal
- Make the before/after slider examples the first thing users see on the homepage.
- Keep the current hero headline and CTAs, but move that copy section directly below the slider showcase.
- Replace the homepage example references with the updated image assets already present in `apps/web/public/images/examples`.
- Remove the stale top-of-page comparator panel that still references deleted assets.

## Recommended Approach
1. Reorder the homepage in `apps/web/src/app/page.tsx` so `BeforeAfterShowcase` renders before the current hero copy.
2. Convert the current hero section into a copy-only section by removing the `HeroExamplePanel` import and right-column usage.
3. Update `apps/web/src/components/marketing/BeforeAfterShowcase.tsx` to use the updated example image pairs and any revised titles/context needed for those assets.
4. Adjust `BeforeAfterShowcase` section classes so it works cleanly as the first section under the header without an unwanted top divider.
5. Delete `apps/web/src/components/marketing/HeroExamplePanel.tsx` if it has no remaining usage after the homepage update.

## Files To Modify
- `apps/web/src/app/page.tsx`
- `apps/web/src/components/marketing/BeforeAfterShowcase.tsx`
- `apps/web/src/components/marketing/HeroExamplePanel.tsx` (remove if unused)

## Implementation Notes
- Keep the change minimal: do not introduce a shared example-data module unless both showcase components still need the same content after the layout change.
- Since the user wants the slider previews to lead the page, preserve the current showcase interaction model and card layout unless visual spacing needs minor adjustment after reordering.
- The current showcase uses `section-block section-divider`; remove or conditionally avoid the divider when it becomes the first section.
- The current hero uses `lg:grid-cols-[1.1fr_0.9fr]`; simplify that layout once the right-side panel is removed so the copy block does not inherit unnecessary grid structure.
- Prefer the updated assets already present in `apps/web/public/images/examples/{before,after}` over the old `person/fruit/object` trio if those no longer represent the intended examples.
- Remove the broken references to deleted assets (`shoe`, `headphones`, `cup`) by deleting the hero comparator component or fully replacing its data if it must stay.

## Verification
1. Run `bun run dev:web` from the repo root.
2. Desktop visual check:
- The slider showcase is the first homepage section visible below the header.
- There is no awkward top border/divider above the first section.
- The headline/CTA hero appears directly below the showcase.
- The hero still feels balanced after removing the comparator column.
3. Mobile visual check:
- The showcase cards stack cleanly and remain readable.
- The hero copy and CTAs wrap cleanly.
4. Interaction check:
- Each before/after slider still works with pointer drag.
- Keyboard controls (`ArrowLeft`, `ArrowRight`, `Home`, `End`) still work.
5. Asset sanity check:
- No missing images render on the homepage.
- Browser console/network show no 404s for deleted example assets.

## Expected Outcome
- The homepage opens with the stronger interactive before/after proof section.
- The user’s updated example images are the ones shown on the homepage.
- The old comparator panel is removed, eliminating stale references and simplifying the hero.
