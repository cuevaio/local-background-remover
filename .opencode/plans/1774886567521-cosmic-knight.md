# Dub-Inspired Web Rebuild Plan (`apps/web`)

## Objective
- Rebuild all of `apps/web` with a Dub.co-inspired visual system (neutral grid, strong type hierarchy, thin border structure, semantic accents, restrained motion).
- Preserve all existing product behavior, route surfaces, and API semantics.

## Non-Negotiable Constraints
- Keep route paths and content purposes intact for `/`, `/pricing`, `/downloads`, `/thank-you`.
- Keep checkout/install/release/license APIs behaviorally unchanged (`/api/checkout`, `/install`, `/api/releases/latest`, `/releases/[tag]/[asset]`, `/api/license/*`).
- Keep installer command URL unchanged: `https://local.backgroundrm.com/install`.
- Preserve SEO surface area (`metadata`, `sitemap`, `robots`, OG/Twitter routes).

## Implementation Plan

### Phase 1: Foundation (tokens, typography, layout rails)
- Replace current global visual foundation with Dub-like semantic tokens in `apps/web/src/app/globals.css`.
- Define a strict typography system (display/headline/body/meta/mono roles) and map to reusable utility classes.
- Introduce shared layout rails: max-width container rhythm, section spacing system, border grid lines, subtle texture utilities.
- Update global shell styling in `apps/web/src/app/layout.tsx` (header/navigation density, sticky treatment, CTA emphasis).

Files:
- `apps/web/src/app/globals.css`
- `apps/web/src/app/layout.tsx`

### Phase 2: Primitive reskin (no API changes to component props)
- Reskin shared UI primitives to consume semantic tokens and consistent motion/interaction states.
- Normalize radius, border, shadow, focus ring, and hover/active behavior across primitives.
- Keep existing component interfaces stable so page logic does not need behavioral rewrites.

- Run: `cd apps/web && bun run dev`

  - `/pricing`
  - `/downloads`
  - `/thank-you`
- Check hierarchy, spacing rhythm, contrast, button states, focus rings, sticky header, and sticky CTA behavior.

### Flow and Contract Regression
- Checkout trigger still posts `{ kind: "app" | "cli" | "both" }` to `/api/checkout` and redirects via returned `url`.
- `thank-you` flow and install guidance remain intact.
- Installer endpoint still serves script via `/install` and command remains `curl -fsSL https://local.backgroundrm.com/install | bash`.
- Release and license endpoints continue existing response semantics.

### SEO/Indexing
- Confirm canonical/title/description and JSON-LD still render on key pages.
- Confirm OG/Twitter image routes resolve.
- Confirm `sitemap.xml` and `robots.txt` outputs remain valid.

## Done Criteria
- Visual system is coherent and token-driven across all `apps/web` surfaces.
- No route/path changes, no checkout/install/license contract regressions.
- All key pages are responsive and production-build clean.
