# Plan: SEO Audit + Ranking Improvements for `apps/web` (TypeScript)

## Goal
- Improve crawlability, index quality, relevance, and SERP presentation for global English queries centered on **"offline background remover"**.
- Concentrate ranking signals on core pages: `/`, `/pricing`, `/downloads`.
- Deindex utility pages/endpoints (`/thank-you`, `/install`) so they do not compete with landing pages.
- Note: no one can guarantee "#1 in Google"; this plan maximizes ranking probability and conversion impact.

## Confirmed Direction
- Primary keyword theme: offline background remover.
- Target market: global English.
- Indexing policy: noindex utility/conversion pages.
- Web app is TypeScript-first (`.ts`/`.tsx`, strict mode) and all SEO work should preserve typed patterns.
- Product invariants to preserve:
  - public install URL remains `https://local.backgroundrm.com/install`
  - downloads remain public
  - license-gated runtime behavior remains unchanged

## Phase 1 - Technical SEO Foundation (Highest Impact)
1. Add crawl/index control routes.
   - Add `apps/web/src/app/robots.ts`.
   - Add `apps/web/src/app/sitemap.ts`.
   - Include only canonical indexable pages in sitemap (`/`, `/pricing`, `/downloads`).
2. Remove indexation noise from utility endpoints/pages.
   - Update `apps/web/src/app/install/route.ts` with `X-Robots-Tag: noindex, nofollow, noarchive`.
   - Add page-level noindex metadata to `apps/web/src/app/thank-you/page.tsx`.
   - Remove `/thank-you` from top nav in `apps/web/src/app/layout.tsx`.
3. Eliminate duplicate query URL risk from checkout success flow.
   - Update `apps/web/src/app/api/checkout/route.ts` default success URL from `/pricing?success=1` to `/thank-you`.

## Phase 2 - Metadata + Canonical Architecture
1. Upgrade global metadata in `apps/web/src/app/layout.tsx`.
   - Add `metadataBase`, title template, default description, default robots, Open Graph, and Twitter config.
   - Keep types explicit with `Metadata` and typed helper return values.
2. Add page-specific metadata for intent alignment.
   - Update `apps/web/src/app/page.tsx` (home intent).
   - Update `apps/web/src/app/pricing/page.tsx` (commercial intent).
   - Update `apps/web/src/app/downloads/page.tsx` (download/install intent).
   - Update `apps/web/src/app/thank-you/page.tsx` (noindex metadata).
3. Add explicit canonical URLs for each indexable page via App Router metadata.

## Phase 3 - On-Page SEO + Internal Linking
1. Improve heading hierarchy and semantic clarity.
   - Ensure one clear H1 per page and logical H2/H3 progression.
2. Strengthen keyword relevance naturally (no stuffing).
   - Refine titles/descriptions/copy around offline, local, private, on-device background removal.
3. Improve internal linking on-page (not just nav).
   - Cross-link `/`, `/pricing`, `/downloads` with descriptive anchors.
4. Improve CTA text specificity on pricing cards.
   - Replace generic "Buy now" with plan-specific purchase intent labels.

## Phase 4 - Structured Data + SERP Preview Assets
1. Add reusable SEO helpers.
   - Add `apps/web/src/lib/seo.ts` for metadata/schema utilities.
2. Implement JSON-LD where it matches visible content.
   - Home: `SoftwareApplication` + `FAQPage` (from existing FAQ block).
   - Pricing/Downloads: `SoftwareApplication` + `BreadcrumbList` as applicable.
3. Add social preview assets and wire them into metadata.
   - Add `apps/web/public/og/default.png` and page-specific variants as needed.

## Phase 5 - CWV/Performance Hardening for Ranking Support
1. Reduce unnecessary client-side JS on pricing page.
   - Convert `apps/web/src/app/pricing/page.tsx` to server-first markup.
   - Move checkout interaction into `apps/web/src/app/pricing/PricingClient.tsx`.
2. Keep layout/visual effects performant on mobile while preserving design language.
   - Review sticky blur behavior in `apps/web/src/app/globals.css` for low-end devices.

## Verification Plan
1. Local build/runtime checks.
   - `bun run dev:web`
   - `cd apps/web && bun run build`
2. Crawl/index controls.
   - Verify `/robots.txt` includes sitemap and correct allow/disallow rules.
   - Verify `/sitemap.xml` contains only canonical indexable pages.
   - Verify `/thank-you` returns noindex metadata.
   - Verify `/install` includes `X-Robots-Tag` noindex header.
3. Metadata/canonical correctness.
   - Confirm unique `title`, `meta description`, and canonical on `/`, `/pricing`, `/downloads`.
   - Confirm OG/Twitter tags resolve correctly.
4. Structured data validation.
   - Validate with Google Rich Results Test for all indexable pages.
5. CWV/SEO quality checks.
   - Run Lighthouse (desktop + mobile) and target SEO >= 90 on core pages.
   - Use browser DevTools to confirm no large hydration regressions on pricing.

## 30/60/90 Day Measurement Plan
- 0-30 days: submit sitemap in GSC, request indexing on updated pages, monitor index coverage and excluded utility URLs.
- 31-60 days: track impressions/CTR/avg position for offline background remover keyword cluster; refine snippets based on CTR.
- 61-90 days: evaluate top-10 term growth, organic landings to `/pricing`, and checkout-start conversion from organic traffic.
